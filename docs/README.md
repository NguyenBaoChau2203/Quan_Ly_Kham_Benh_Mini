# docs

Tài liệu chính:

- `Team_Workflow.md`: hướng dẫn cả nhóm đọc gì, làm gì, và tránh conflict khi code song song
- `Master_Plan.md`: chuẩn triển khai đã chốt (spec v1_3 + các điểm vá A–F)
- `tasks/`: phân công theo từng thành viên

Quy tắc:

- Thành viên mới/pull code mới đọc `Team_Workflow.md` trước khi bắt đầu sửa file.
- Khi có thay đổi yêu cầu/luồng/trạng thái/SP, cập nhật vào `Master_Plan.md` trước, rồi mới code.
